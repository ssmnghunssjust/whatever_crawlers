# Contributing to MyDataHomes

首先感谢对于**MyDataHomes**项目的关注！

**MyDataHomes**项目一直致力于提供安全的工具帮助个人用户拿回自己的数据，实现数据价值。
您可以通过多种方式来为本项目贡献，本文档会提供关于代码贡献的一些规范指导，但不是强制要求。您可以自由地提出任何修改方案，包括对于本文档的修改。

如果您遇到了关于本项目的设计、使用等方面的问题，可以加入我们的[官方讨论社区](https://mydata.homes)。

## 如何贡献

### Bug报告
Bug是软件项目不可避免的现实问题，我们非常乐意接受来自开发者和使用者的问题反馈。本节将指导您如何提交一份bug报告来协助项目维护人员理解、复现并解决bug。

当您发现软件的异常行为或是代码中的漏洞，请首先搜索是否已经存在同样的问题。如果不存在，请尽可能多地提供关于该问题的细节，并填写[bug报告模板](https://github.com/MyDataHomes/Spider/issues/new?template=bug_report.md)。


如果您觉得某些bug会伤害到您的隐私或是带来安全风险，亦或是bug的公开对项目以及其他开发者、使用者造成不好的影响，请遵循[responsible disclosure](https://en.wikipedia.org/wiki/Responsible_disclosure)，并直接将bug报告发送至我们的邮箱contact@trustmirror.com, 我们的技术团队会及时回复。

### 新的爬虫

在爬虫的开发方面，我们一方面希望不断扩充可采集的数据源数量，另一方面希望不断完善现有爬虫的质量，包括性能、兼容性、易用性能质量属性。

当前我们接受如下两方面的工作：
1. 增加更多的数据源爬虫, 具体步骤如下:
  - 在Spiders目录中新建一个数据源目录
  - 提交爬虫源码 + ReadMe + 数据格式定义文件或示例
  - 给前端工具界面（tools/main.py）增加对应的数据源按钮

2. 完善已有数据源下载函数,使得其能够下载更为完整的用户数据。

(为保持统一, 要求 Python3 开发,数据存为 JSON 格式)
